memcache基础

Posted on 2020-04-07 In memcache

  1. set() 增/改
    语法: set key msg cache_time length
    key: 不能超过250字符
    msg: key 的额外信息 (整型参数)
    cache_time(/s): 0-理论永久 (最大30天)
    length: 数字类型,单位字节

  2. get()
    语法: get key

  3. replace()
    语法: replace key 是否压缩(0/1) cache_time/s length/字节
    只能修改不能添加

  4. add()
    语法: add key 是否压缩 cache_time/s length/字节
    只能添加不能修改

  5. delete()
    语法: delete key

  6. incr() decr()
    语法: ``` incr key nums 

    decr key nums  ```

    返回增长后的接口 ,key必须存在

  7. flush_all

set() add() 区别

add添加key时如果存在会返回false  
set会直接覆盖 相当于add 和replace的载体

1.innodb myisam(mysql8 已取消)之间的区别

1.事务和外键

innodb支持事务,和四种隔离级别,回滚,崩溃修复能力.
如果有大量的insert或update操作该使用innodb,可以提高多用户并发的性能.

myisam有高速的存储和检索能力,和全文搜索(新版的innodb已经支持).大量的select可以用myisam

2.锁

msql支持行锁,页锁,表锁

myisam支持表级锁

innodb支持行级锁,但不绝对.一个sql语句中不确定扫描的范围,innodb会锁全表
