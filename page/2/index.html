存储引擎

1.innodb myisam(mysql8 已取消)之间的区别

1.事务和外键

innodb支持事务,和四种隔离级别,回滚,崩溃修复能力.
如果有大量的insert或update操作该使用innodb,可以提高多用户并发的性能.

myisam有高速的存储和检索能力,和全文搜索(新版的innodb已经支持).大量的select可以用myisam

2.锁

msql支持行锁,页锁,表锁

myisam支持表级锁

innodb支持行级锁,但不绝对.一个sql语句中不确定扫描的范围,innodb会锁全表

redis 事务不支持回滚

秒杀逻辑

用户并发->扣库存->生成订单->支付

可能出现问题

先扣库存解决超发

订单过期解决恶意订单

连接redis

$redis = new redis();
$redis->connect('localhost',6379)

获取库存存入redis

$stock = select 'stock' form goods where goods_id=1
$stock_left = $reids->get('stock_left');  ##此时stock_left默认为0

逻辑处理

if($stock>$stock_left){
   $redis->watch('stock_left'); 监视变量 stock_left
   $redis->multi();  开启事务

   $redis->incr('stock_left'); 让stock_left + 1
   $redis->exec(); 提交事务

   echo $reids->get('stock_left');
}
